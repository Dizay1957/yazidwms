## Recommended Settings

- Pin the repository on the GitHub profile.
- Add a social preview image after screenshots are captured.
- Enable branch protection after the first CI run passes.
- Require the CI workflow on pull requests targeting `main`.
- Add a demo URL later if the project is deployed.
- Create a `v1.0.0` release after the documentation and screenshots are complete.
- Choose a license before accepting external contributions. MIT is common for portfolio projects, but licensing is an owner decision.
- Keep `.env` files out of Git and rotate any secret that was ever exposed publicly.
